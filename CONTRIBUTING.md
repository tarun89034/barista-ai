# Contribution Guidelines

Thank you for considering contributing to the **barista-ai** project! We welcome contributions from everyone. Below are the guidelines to help you get started.

## Reporting Bugs
If you encounter any bugs, please report them by following these steps:
1. Check the [issue tracker](https://github.com/tarun89034/barista-ai/issues) to see if the bug has already been reported.
2. If not, create a new issue with a clear and descriptive title and provide the following details:
   - Steps to reproduce the bug
   - Expected behavior
   - Actual behavior
   - Screenshots or additional information, if applicable

## Suggesting Features
We are always open to suggestions for new features! To suggest a feature:
1. Check if it’s already suggested in the [issues](https://github.com/tarun89034/barista-ai/issues).
2. If it isn’t, please create a new issue with a descriptive title and details about your feature idea, including its benefits.

## Submitting Pull Requests
To contribute code changes:
1. Fork the repository and clone it on your machine.
2. Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b my-feature-branch
   ```
3. Make your changes and commit them with descriptive messages:
   ```bash
   git commit -m "Add my feature"
   ```
4. Push your branch to your fork:
   ```bash
   git push origin my-feature-branch
   ```
5. Open a pull request to the **main** branch of the original repository. Describe your changes concisely.

## Coding Standards
- Follow the project's coding style guidelines.
- Use meaningful variable and function names.
- Write clear and concise comments.

## Testing Guidelines
- Ensure to run existing tests and pass them.
- If applicable, write new tests for your code.
- Use a consistent testing framework as defined in the repository.

## Areas for Contribution
- Bug fixing
- Adding new features
- Improving documentation
- Enhancing tests

We appreciate your contributions to **barista-ai**! Thank you for helping make the project better.
